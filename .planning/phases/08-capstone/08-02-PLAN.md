## Common Mistakes

### Planning Phase
- Designing too many endpoints before building any (start with auth + one resource)
- Skipping the schema design exercise (leads to migration headaches later)

### Implementation Phase
- Putting business logic in route handlers instead of services (Module 008 pattern)
- Using plain dict responses instead of Pydantic response models
- Forgetting to add created_at/updated_at timestamps to models

### Testing Phase
- Only testing happy paths (test 400/401/403/404 responses too)
- Not using fixtures for test data isolation
- Testing implementation details instead of behavior

### Deployment Phase
- Hardcoding secrets instead of using environment variables
- Missing health check endpoint
- Not configuring CORS for frontend integration
