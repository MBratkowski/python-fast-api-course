Anti-pattern: trusting file extensions. Anti-pattern: reading entire file into memory for size check. Code example: validation function that checks type and size.
    3. `03-local-storage.md` -- Saving files with shutil.copyfileobj (streaming, memory-efficient), Path operations, unique filename generation (UUID prefix), serving static files with StaticFiles mount, cleanup strategies. Anti-pattern: using await file.read() for large files. Code example: upload + save + serve endpoint.

Mobile parallel: AWS Amplify Storage (iOS), Firebase Storage (Android). Code example: S3 upload function. Note: exercises use local storage; S3 covered in theory and project.
    5. `05-presigned-urls.md` -- What presigned URLs solve (direct client-to-S3 upload), generate_presigned_url for GET and PUT, expiration configuration, security benefits (server never touches file bytes). Mobile parallel: Firebase Storage download URLs. Code example: endpoint that returns presigned upload URL.
    6. `06-image-processing.md` -- Pillow basics: open, resize, thumbnail, format conversion, EXIF handling. Processing pipeline: upload -> validate -> resize -> save. Generating thumbnails. Mobile parallel: UIImage processing (iOS), Bitmap operations (Android). Code example: upload endpoint that creates thumbnails.

    - Emphasize: mobile developers consume WebSocket connections; now they build the server side

    **Theory files (6):**
    1. `01-websocket-vs-http.md` -- HTTP request/response vs WebSocket persistent connection, when to use each (polling vs SSE vs WebSocket decision tree), WebSocket handshake (HTTP upgrade), bidirectional communication. Mobile parallel: URLSessionWebSocketTask (iOS), OkHttp WebSocket (Android). ASCII diagram showing HTTP vs WebSocket lifecycle.
    2. `02-fastapi-websocket.md` -- @app.websocket decorator, WebSocket accept/receive/send lifecycle, receive_text() and send_text(), receive_json() and send_json(), WebSocketDisconnect exception handling, try/except pattern for connection loop. Code example: echo server WebSocket endpoint.
    3. `03-connection-manager.md` -- ConnectionManager class pattern (from FastAPI docs): active_connections list, connect() (accept + append), disconnect() (remove), send_personal_message(), broadcast(). Why a manager: multiple clients, message routing, cleanup. Code example: full ConnectionManager with chat endpoint.
    4. `04-broadcasting-rooms.md` -- Room-based messaging: dict[str, list[WebSocket]] for rooms, join_room/leave_room methods, broadcast_to_room. Use cases: chat rooms, game lobbies, topic subscriptions. Mobile parallel: Firebase Realtime Database rooms, Combine Publishers. Code example: ConnectionManager with room support.
    5. `05-websocket-authentication.md` -- Challenge: no HTTP headers in WebSocket handshake. Solutions: query parameter token (?token=xxx), cookie-based auth, first-message auth protocol. Using Depends() with WebSocket. WebSocketException with WS_1008_POLICY_VIOLATION. Code example: JWT auth via query parameter (per research, referencing Phase 3 PyJWT decision).
    6. `06-scaling-redis-pubsub.md` -- Single-server limitation: ConnectionManager is in-process. Redis Pub/Sub for multi-server broadcasting. Pattern: publish to Redis channel, each server subscribes and broadcasts to local connections. Mobile parallel: APNs/FCM push notification fan-out. Code example: Redis Pub/Sub integration with ConnectionManager.
