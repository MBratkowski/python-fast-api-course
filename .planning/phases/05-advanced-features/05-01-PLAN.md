    5. `05-retries-error-handling.md` -- autoretry_for with exception types, retry_backoff=True (exponential), retry_backoff_max, retry_jitter (thundering herd prevention), max_retries, manual retry with self.retry(exc=exc, countdown=N). bind=True for self access. Mobile parallel: WorkManager.setBackoffPolicy().
    6. `06-scheduled-tasks-celery-beat.md` -- Celery Beat for periodic tasks, beat_schedule configuration, crontab() and timedelta schedules, timezone handling. Mobile parallel: BGAppRefreshTaskRequest (iOS), PeriodicWorkRequest (Android). Include example: scheduled database cleanup task.

    **Module README (014-caching-redis/README.md):**
    - Why This Module, What You'll Learn, Mobile Developer Context
    - Comparison table: iOS NSCache / Android LruCache / Python+Redis
    - Quick Assessment, Time Estimate, Key Differences from Mobile
    - Emphasize: mobile caches are in-process memory; server caches are external services shared across instances

    **Theory files (6):**
    1. `01-why-cache.md` -- Cache benefits (latency, load reduction, cost), cache hit/miss concepts, where caching fits in request lifecycle. Mobile parallel: NSCache for images (iOS), LruCache (Android). Include latency comparison: DB query ~5ms vs Redis ~0.5ms.
    2. `02-redis-setup.md` -- Redis installation via Docker Compose, redis-py library (NOT aioredis -- deprecated, merged into redis-py), sync vs async clients (redis.Redis vs redis.asyncio), connection pooling with from_url(), FastAPI lifespan for connection lifecycle. Include docker-compose snippet.
    3. `03-caching-patterns.md` -- Cache-aside (look-aside) pattern: check cache -> miss -> fetch DB -> store cache. Write-through vs write-behind. Read-through. When to use each. Code example: cache-aside with FastAPI dependency. Mobile parallel: URLCache (iOS), OkHttp cache (Android).
    4. `04-ttl-expiration.md` -- SETEX and EXPIRE commands, choosing TTL values (hot data: short TTL, reference data: long TTL), memory management with maxmemory and eviction policies (allkeys-lru). Anti-pattern: caching without TTL.
    5. `05-cache-invalidation.md` -- "Two hard problems in CS" intro. Strategies: TTL-based (passive), explicit DELETE on write (active), tag-based invalidation (group keys). Pattern: always invalidate on create/update/delete. Consistent key naming: `entity:{id}`. Code examples for each strategy.
    6. `06-redis-data-structures.md` -- Strings (simple cache), Hashes (object fields), Sets (unique collections), Sorted Sets (leaderboards, ranked data), Lists (queues). When to use each. Code examples with redis-py commands. Mobile parallel: UserDefaults/SharedPreferences store simple k-v; Redis handles complex structures.

    Every theory file MUST include:
    - "Why This Matters" section with mobile-dev analogy
    - Code examples (working Python code)

    - "Key Takeaways" bullet list

TODO stubs. Inline pytest tests verify TTL behavior with fakeredis (use time-based assertions or fakeredis time controls).
